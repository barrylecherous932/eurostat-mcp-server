# 0.1.6 — 2026-05-24

## Fixed

- **`eurostat_query_dataset`** — unfiltered queries that return > 5,000 observations now cap at 5,000 rows and set `truncated: true` on the result, preventing the ~355MB crash seen with broad dataset requests ([#4](https://github.com/cyanheads/eurostat-mcp-server/issues/4)).
- **`eurostat_query_dataset`** — `since_period`/`until_period` combined with `last_n_periods` now raises a `conflicting_params` error instead of silently ignoring the date range ([#5](https://github.com/cyanheads/eurostat-mcp-server/issues/5)).
- **`eurostat_browse_themes`** — `notFound` error now includes `data.reason: "not_found"` so agents can branch on the error code ([#6](https://github.com/cyanheads/eurostat-mcp-server/issues/6)).
- **TOC parser** — `dataStart`/`dataEnd` values are trimmed before the falsy check, so entries that carry only whitespace no longer surface as `" "` in folder results ([#7](https://github.com/cyanheads/eurostat-mcp-server/issues/7)).
