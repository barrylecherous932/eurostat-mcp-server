# 0.1.8 — 2026-05-28

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.9` → `^0.9.13` — picks up four framework releases:
  - **`MCP_HTTP_MAX_BODY_BYTES`** — configurable inbound request-body cap (default 1 MiB); oversized requests rejected with HTTP 413 before parsing ([#157](https://github.com/cyanheads/mcp-ts-core/issues/157))
  - **HTTP session-init gate** — stateful HTTP mode now rejects requests without `Mcp-Session-Id` with HTTP 400, preventing uninit session minting ([#154](https://github.com/cyanheads/mcp-ts-core/issues/154))
  - **Expected client error log levels** — 401/403/400/404 errors use `logger.warning` instead of the full error-handler pipeline, reducing stack-trace noise in logs ([#158](https://github.com/cyanheads/mcp-ts-core/issues/158))
  - **`GET /mcp` keywords** — `package.json` `keywords` now surfaced on the status JSON response ([#165](https://github.com/cyanheads/mcp-ts-core/issues/165))
- **`landing.requireAuth: false`** — explicitly opts the landing page into public inventory (keeps current behavior against the 0.9.13 `jwt`/`oauth` default-auth change; this server uses `MCP_AUTH_MODE=none` but the explicit opt-in future-proofs the config).
- **`@biomejs/biome`** `^2.4.15` → `^2.4.16`
- **Skills synced** to mcp-ts-core 0.9.13 — `api-canvas`, `api-config`, `design-mcp-server`, `git-wrapup`, `polish-docs-meta`, `release-and-publish`, `report-issue-framework`; `migrate-mcp-ts-template` removed; `code-simplifier` added.
- **Plugin metadata scaffolded** — `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json` + `mcp.json` added.
