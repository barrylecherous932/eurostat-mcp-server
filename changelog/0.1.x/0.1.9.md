# 0.1.9 — 2026-05-30

## Added

- **`enrichment` block** on `eurostat_search_datasets`, `eurostat_browse_themes`, and `eurostat_query_dataset` — surfaces agent-facing metadata in a typed `enrichment` block reaching both `structuredContent` JSON and the `content[]` markdown trailer, so both Claude Code and Claude Desktop clients receive the same signal.
  - `eurostat_search_datasets`: `query` (echoed search terms), `totalMatches` (true count before limit)
  - `eurostat_browse_themes`: `itemCount` (items returned), `themeCode` (folder browsed, when set)
  - `eurostat_query_dataset`: `appliedFilters` (dimension filters, geo level, time range), `notice` (truncation guidance when the 5,000-row cap is hit)

## Changed

- **`eurostat_search_datasets` `format()`** — text header changed from `N total matches (showing N)` to `Showing N results`; the true result count now surfaces in the `enrichment` block.
- **`eurostat_search_datasets` `structuredContent`** — `totalMatches` moved from the `output` object to `enrichment`; value and presence preserved in both channels, key path changed (`result.totalMatches` → `result.enrichment.totalMatches`).
- **`@cyanheads/mcp-ts-core`** `^0.9.13` → `^0.9.16` — picks up enrichment builder, `ctx.enrich()` / `ctx.enrich.notice()`, enrichment schema support, and skill sync.
